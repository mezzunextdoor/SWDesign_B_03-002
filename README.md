다음의 사용자 요구사항을 기반으로 클래스 다이어그램을 설계하고, 예약 기능을 구현하시오.
- 클래스명, 변수명, 함수명은 한글로 정의할 것. 반드시 UML 규칙을 따를 것.
- 속성의 가시성을 private 으로 정의할 것. 행위의 가시성을 public 으로 정의할 것.
- 속성의 데이터 타입과 행위의 반환 데이터 타입은 아래의 요구사항을 분석하여 정의할 것.
- 다중성을 반드시 설계할 것.
- "ReservationUI" boundary 클래스를 추가하여 설계할 것.
- '예약' 기능만 Java로 구현하면 되며, 참조하는 다른 클래스의 행위는 구현하지 않아도 됨.

본 리조트 객실 예약 시스템은 고객정보(id, 암호, 성명)와 객실정보(객실id, 평수, 객실가격)를 기반으로
예약정보(예약id, id, 객실id, 숙박일자, 숙박기간, 총비용)에 의해 예약할 수 있다.
예약정보는 예약, 취소, 예약조회 기능을 제공한다.
추가적으로 본 시스템에서는 고객등록, 고객조회, 고객인증 기능과 객실등록, 객실조회, 객실가격조회 기능을 제공한다.

예약 시 반드시 고객인증과 객실조회를 해야 하며,
	예약id는 id, 객실id, 숙박일자를 더하여("+") 생성한다.
	숙박일자는 시스템 날짜를 자동으로 받지 않고 입력
	총비용은 객실가격 * 숙박기간으로 계산하여 저장한다.
	예약id, id, 객실id, 숙박일자, 숙박기간, 총비용을 속성에 저장한다. 예약이 완료되면 총비용을 출력한다.

취소 시 반드시 예약조회와 고객조회를 해야 한다.
	예약조회는 예약id에 의해 조회가 가능하다.
